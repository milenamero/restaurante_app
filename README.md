## ¿De qué trata este proyecto?
Hemos creado este sistema para organizar la gestión de clientes y productos de un restaurante. Lo hicimos aplicando Programación Orientada a Objetos (POO) porque nos ayuda a que el código sea más ordenado y fácil de entender.

## ¿Cómo está organizado?
Dividimos todo en carpetas para que no sea un desorden:
- `modelos/`: Aquí están las clases de `Producto` y `Cliente`.
- `servicios/`: Aquí pusimos la lógica del `Restaurante`.
- `main.py`: Es el archivo principal para ejecutar el programa.

## ¿Cómo correrlo?
Es súper fácil, solo necesitas tener instalado Python. Abre tu terminal en la carpeta del proyecto y escribe:
`py main.py`

## ¿Por qué lo hicimos así? Modularidad
La verdad, usar la modularidad nos ayudó un montón. En lugar de tener todo el código mezclado en un solo lugar, al separarlo en módulos es mucho más sencillo hacer cambios o arreglar errores sin que se dañe todo el programa. Así es más fácil trabajar y mantener el proyecto ordenado.
